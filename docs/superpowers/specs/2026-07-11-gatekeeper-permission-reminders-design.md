# 出发守门权限提醒显示逻辑

## 目标

出发守门页底部只展示尚未完成、需要用户处理的离线资源或权限提醒。已经就绪的项目不再占据前端空间；所有项目均就绪时，整张“离线与权限”卡隐藏。

## 用户情境与主动作

用户即将出发，注意力应集中在补给、装备和“接受风险并出发”上。权限区只在确实阻止出发时出现，唯一作用是指出尚未完成的准备事项，不与主动作竞争。

## 状态与显示规则

- GPX / 路线资源未就绪：显示离线资源提醒与完整性说明。
- 定位未授权：显示定位权限提醒。
- 通知未授权：显示通知权限提醒。
- 任一项目已就绪：不显示该项目的成功行。
- 三项全部就绪：不渲染整张“离线与权限”卡。
- 出发按钮的放行条件保持不变，仍要求必选装备、离线资源、定位和通知全部就绪。

## 实现设计

在当前承载出发守门逻辑的 `BudgetCardView.swift` 中增加一个小型、纯状态的 readiness 模型，把三个布尔输入转换为待提醒事项列表。视图只遍历该列表，列表为空时不插入权限卡。该模型不依赖 Core Location、通知中心或 SwiftUI 环境对象，因此可以用单元测试覆盖。

现有权限请求流程保持不变：页面出现时仍请求定位和通知授权。视图显式订阅嵌套的定位、通知和离线资源对象；状态改变后重新计算待提醒列表并隐藏已完成项。

## 测试与验收

- 全部就绪时，待提醒列表为空。
- 只有定位未授权时，仅返回定位提醒。
- 只有通知未授权时，仅返回通知提醒。
- 只有离线资源未就绪时，仅返回离线资源提醒。
- 多项未就绪时，按离线资源、定位、通知的稳定顺序返回。
- 现有测试与项目构建继续通过。

## 设计约束

本次不新增卡片、成功徽章、渐变、Glass 或动画。删除已授权的绿色成功行和无意义的常驻卡片，让出发页在准备完成后更简洁，并保持风险放行规则不变。
